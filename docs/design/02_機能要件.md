# 2. 機能要件

## 2.1 ユーザーロール定義

### ロール一覧

| ロール | 説明 | 権限レベル |
|--------|------|-----------|
| 監督（Manager） | チームを管理する責任者 | 高 |
| 選手（Player） | チームに所属するメンバー | 標準 |
| ゲスト（未認証） | 未ログインユーザー | 低 |

## 2.3 機能詳細仕様

### C-001: ユーザー登録

```mermaid
sequenceDiagram
    participant User as ユーザー
    participant App as アプリ
    participant Auth as Supabase Auth
    participant DB as Database

    User->>App: 登録フォーム入力
    App->>App: 入力バリデーション
    App->>Auth: signUp(email, password)
    Auth-->>App: 認証ユーザー作成
    App->>DB: usersテーブルにINSERT
    DB-->>App: ユーザー情報
    App-->>User: 登録完了・チーム選択へ遷移
```

**入力項目:**
- メールアドレス（必須）
- パスワード（必須、6文字以上）
- ユーザー名（必須）
- ロール選択（監督/選手）

**バリデーション:**
- メールアドレス形式チェック
- パスワード長チェック（6文字以上）
- 必須項目チェック

### M-003: 試合作成

```mermaid
sequenceDiagram
    participant Manager as 監督
    participant App as アプリ
    participant DB as Database

    Manager->>App: 試合情報入力
    App->>App: 入力バリデーション
    App->>DB: gamesテーブルにINSERT
    DB-->>App: 試合ID
    App->>DB: team_membersから選手取得
    DB-->>App: 選手リスト
    App->>DB: attendancesに全選手分INSERT
    DB-->>App: 出欠レコード作成完了
    App-->>Manager: 作成完了・試合詳細へ遷移
```

**入力項目:**
- 対戦相手（必須）
- 試合日（必須）
- 試合時間（必須）
- 場所（任意）
- 備考（任意）

### M-009: ポジション設定

```mermaid
sequenceDiagram
    participant Manager as 監督
    participant App as アプリ
    participant Field as SoccerField
    participant DB as Database

    Manager->>App: ポジション設定開始
    App->>DB: 出席選手・ポジションマスタ取得
    DB-->>App: データ返却
    App->>Field: フィールド表示
    
    alt フォーメーション適用
        Manager->>App: フォーメーション選択
        App->>Field: 自動配置
    else 個別調整
        Manager->>Field: 選手をドラッグ/交換
        Field->>Field: 座標更新
    end
    
    Manager->>App: 保存
    App->>DB: attendancesを更新
    DB-->>App: 保存完了
    App-->>Manager: 成功メッセージ
```

**機能詳細:**

1. **フォーメーション選択**
   - プリセット: 4-4-2, 4-3-3, 3-5-2, 4-2-3-1, 3-4-3
   - 選択時に自動で選手を配置

2. **編集モード**
   - **交換モード**: 2人の選手をクリックして位置交換
   - **ドラッグモード**: 選手を自由にドラッグして微調整

3. **スタメン/サブ管理**
   - 最大11人までスタメン配置可能
   - 控え選手リストで管理

### P-002: 出欠回答

```mermaid
stateDiagram-v2
    [*] --> 未回答
    未回答 --> 出席: 出席ボタン押下
    未回答 --> 欠席: 欠席ボタン押下
    未回答 --> 未定: 未定ボタン押下
    出席 --> 欠席: 欠席ボタン押下
    出席 --> 未定: 未定ボタン押下
    欠席 --> 出席: 出席ボタン押下
    欠席 --> 未定: 未定ボタン押下
    未定 --> 出席: 出席ボタン押下
    未定 --> 欠席: 欠席ボタン押下
```

**ステータス値:**
- `participate`: 出席
- `absent`: 欠席
- `unanswered`: 未回答/未定

## 2.4 ビジネスルール

### チーム管理ルール

| ルールID | ルール内容 |
|----------|-----------|
| BR-001 | 1人の監督は複数のチームを作成・管理できる |
| BR-002 | 1人の選手は複数のチームに所属できる |
| BR-003 | チーム加入には監督の承認が必要 |
| BR-004 | 却下された申請は再申請可能 |

### 試合管理ルール

| ルールID | ルール内容 |
|----------|-----------|
| BR-101 | 試合作成時、全所属選手に出欠レコードが自動作成される |
| BR-102 | 試合削除時、関連する出欠レコードも削除される |
| BR-103 | 出欠は試合当日まで変更可能 |

### ポジション設定ルール

| ルールID | ルール内容 |
|----------|-----------|
| BR-201 | フィールドに配置できるのは最大11人 |
| BR-202 | 出席（participate）の選手のみ配置可能 |
| BR-203 | ポジション設定は監督のみ可能 |
| BR-204 | フォーメーション適用は選手数に応じて調整 |
